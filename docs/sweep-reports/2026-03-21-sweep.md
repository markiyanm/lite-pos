## P0 CRITICAL (1)

### 9. No transaction wrapper for order creation — partial writes on failure
**Files:** `src/routes/pos/+page.svelte:290-389`, `src/routes/pos/+page.svelte:77-118`, `src/routes/orders/[id]/+page.svelte:214-263`
**Evidence:** `handlePaymentComplete` performs createOrder → addOrderItem loop → adjustStock loop → addPayment loop → completeOrder as separate SQL calls. If any step fails mid-way (e.g. stock adjustment throws on item 3), the order row exists, some items are persisted, stock is partially deducted, but later items/payments are missing. No BEGIN/COMMIT/ROLLBACK. Same pattern in `handleSaveDraft` and refund processing.
**Slug:** `wrap-order-creation-in-a-db-transaction-to-prevent-partial-writes`

## P1 IMPORTANT (8)

**Evidence:** shadcn-svelte Select expects `value: string[]` but gets `string`. Card-present terminal selection broken.

### 2. PaymentModal: `bind:devices` on non-bindable prop
**File:** `src/lib/components/pos/PaymentModal.svelte:563`
**Evidence:** `devices` not declared `$bindable()` in DeviceSelector.
**Slug:** `fix-paymentmodal-binddevices-on-non-bindable-deviceselector-prop`

### 3. SettingToggle missing `disabled` prop
**File:** `src/lib/components/settings/SettingToggle.svelte` + settings page (3 places)
**Evidence:** Non-admin users can toggle payment methods and Sola gateway settings.
**Slug:** `add-disabled-prop-to-settingtoggle-to-enforce-admin-only-settings`

### 4. Order number consumed on payment modal cancel
**File:** `src/routes/pos/+page.svelte:73`
**Evidence:** `getNextOrderNumber()` increments DB counter on modal open. Cancel = permanent gap.
**Slug:** `fix-order-number-consumed-on-payment-modal-cancel`

### 10. No admin guard on routes or data-modifying commands
**File:** `src/routes/+layout.svelte:30-34`, all commands in `src/lib/commands/`
**Evidence:** Route guard only checks `isAuthenticated`. A cashier can navigate to /settings, /products/new, /categories, /reports. Command functions (createUser, updateSetting, deleteProduct, etc.) perform zero permission checks.
**Slug:** `add-admin-guard-on-routes-and-data-modifying-commands`

### 11. PINs stored in plaintext
**File:** `src/lib/commands/auth.ts:6,22`
**Evidence:** Column is named `pin_hash` but `createUser` stores raw PIN. Anyone with SQLite file access can read all PINs.
**Slug:** `hash-user-pins-instead-of-storing-plaintext`

### 12. Hardcoded readable AES encryption key in binary
**File:** `src-tauri/src/crypto.rs:10`
**Evidence:** Static AES-256 key compiled into binary as readable ASCII: `LitePOS-SolaGateway-AESKey_2026!`. Extractable via `strings` command. All encrypted Sola API keys can be trivially decrypted.
**Slug:** `replace-hardcoded-aes-encryption-key-with-os-keychain`

### 13. `$derived(() => ...)` used instead of `$derived.by()` in 25+ locations
**Files:** PaymentModal (7 locations), OrderPanel, ProductGrid, all list pages, reports, settings
**Evidence:** In Svelte 5, `$derived(() => expr)` stores the function itself, not the result. Templates invoke the function on every access, defeating reactivity caching. In PaymentModal, `remainingCents()` is called 10+ times in the template, recomputing each time.
**Slug:** `fix-derived-reactivity-replace-derived-with-derivedby-across-codebase`
